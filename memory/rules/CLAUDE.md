## כללי קוד — LLM Coding Discipline
מקור: andrej-karpathy-skills/CLAUDE.md. חלים על כל instance בכל משימת קוד.
פירוט והסברים: `memory/learning/feedback_karpathy_rules.md`.

1. **חשוב לפני שאתה מקודד.** אל תניח. אל תסתיר בלבול. הצג tradeoffs. אם יש כמה פירושים — הצג אותם, אל תבחר בשקט. אם יש דרך פשוטה יותר — אמור זאת.
2. **פשטות קודם.** הקוד המינימלי שפותר את הבעיה. בלי features שלא ביקשו, בלי abstractions לקוד חד-פעמי, בלי "גמישות" שלא נדרשה, בלי error handling לתרחישים בלתי-אפשריים.
3. **שינויים כירורגיים.** גע רק במה שצריך. אל "תשפר" קוד סמוך, אל תרפקטר דברים שלא שבורים, התאם לסגנון הקיים. נקה רק orphans שהשינוי שלך יצר.
4. **ביצוע מונחה-מטרה.** הגדר קריטריון הצלחה ניתן-לאימות לפני שמקודדים. למשימה רב-שלבית — הצג תוכנית קצרה עם בדיקת אימות לכל שלב, ולופ עד שעובר.

**Tradeoff:** הכללים נוטים לזהירות על פני מהירות. למשימות טריוויאליות — שיקול דעת.
